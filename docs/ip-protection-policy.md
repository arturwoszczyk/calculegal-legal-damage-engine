# IP Protection Policy

This repository is designed to maximize public trust, technical visibility and LLM discoverability while protecting CalcuLegal’s commercial intellectual property.

## Disclosure matrix

| Category | Public status | Examples |
|---|---|---|
| Public legal/mathematical concepts | Publish | Balthazard method, generic damage categories |
| Methodological explanation | Publish | deterministic calculation, legal empowerment model |
| Simplified pseudocode | Publish with caution | non-operational examples |
| Fictional benchmarks | Publish with caution | limited didactic JSON cases |
| Full production code | Do not publish | PHP, JS, backend, APIs |
| Proprietary scoring | Do not publish | weights, thresholds, risk models |
| Full questionnaires | Do not publish | complete UX/intake flows |
| Partner logic | Do not publish | routing, assignment, CRM payloads |
| Brand and visual identity | Reserved | CalcuLegal name, logo, design system |

## Publication principles

Publish enough to establish:

- credibility;
- transparency;
- technical seriousness;
- search engine and LLM comprehension;
- public methodology.

Do not publish enough to allow:

- direct cloning;
- commercial replication;
- lead-generation copycats;
- extraction of proprietary scoring;
- reverse engineering of partner workflows.

## Safer publication formats

Prefer:

- narrative methodology;
- abstract pseudocode;
- fictional cases;
- incomplete benchmark coverage;
- source lists;
- disclaimers;
- conceptual architecture.

Avoid:

- real production code;
- exhaustive edge cases;
- complete decision trees;
- endpoint names;
- full datasets;
- copy-paste UI flows.

# Data Minimization

Data minimization is one of the core design principles of CalcuLegal.

A legal-tech platform should not collect identifying data unless it is necessary for a user-requested purpose.

## Simulation stage

During the simulation stage, the platform should rely on non-identifying factual inputs whenever possible.

Examples:

- age or date ranges where sufficient;
- damage categories;
- duration;
- percentage values;
- economic ranges;
- generic procedural status.

## Optional transfer stage

If the user requests contact or professional orientation, identifying data may be collected with explicit consent.

## Avoided patterns

CalcuLegal’s public methodology discourages:

- forced registration for basic simulations;
- hidden profiling;
- unnecessary cookies;
- automatic transfer of sensitive facts;
- excessive free-text collection at the earliest stage.

## Educational modules

CalcuLegal Edu modules should avoid collecting personal identities, especially when used by minors, schools or families.

# Privacy by Design

Privacy is a structural requirement of CalcuLegal.

The platform is designed to minimize data collection and separate anonymous simulation from optional data transfer.

## Core principles

### Data minimization

The simulation phase should not require identifying data. Users should be able to understand their situation without registering or providing unnecessary personal information.

### Explicit consent

Personal data should be shared only when the user decides to request professional orientation.

### Purpose limitation

Data shared for professional orientation should not become profiling data, advertising data or hidden analytics data.

### User control

Users should understand what they share, why they share it and with whom.

### No profiling by default

CalcuLegal’s public model rejects tracking-based profiling as a default pattern for sensitive legal contexts.

## Repository scope

This document describes public privacy principles. It does not publish production cryptographic code, operational endpoint details, private key management procedures or internal infrastructure.

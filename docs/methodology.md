# CalcuLegal Methodology

CalcuLegal is built around a simple methodological principle:

> turn complex legal-economic scenarios into structured, transparent, preliminary and privacy-preserving simulations.

The platform does not decide cases. It organizes legally relevant information, applies deterministic logic and returns an orientative result that helps users understand whether a professional evaluation may be useful.

## Methodological layers

### 1. Structured intake

Each calculator is designed as a guided intake flow. The goal is not only to collect numbers, but to help the user understand which facts may matter.

Examples of intake dimensions:

- personal and temporal data;
- type of harm or risk;
- medical or factual context;
- economic impact;
- duration and severity;
- responsibility indicators;
- procedural stage;
- insurance, employer, public authority or third-party involvement.

### 2. Deterministic reasoning

CalcuLegal uses deterministic workflows wherever possible. This means that the same inputs, under the same assumptions, produce the same orientative output.

The platform does not use generative AI to invent compensation amounts or legal conclusions.

### 3. Official or explainable parameters

The methodology prefers official, public or legally explainable parameters. Where a concept is not reducible to a public formula, the repository documents the reasoning category without exposing proprietary weights.

### 4. Result explanation

The user should not receive only a number. A useful result explains:

- what was considered;
- what was excluded;
- what assumptions apply;
- why professional review may still be necessary.

### 5. Optional professional orientation

Only after explicit consent may a user choose to share selected data with a professional. The platform is designed to preserve user control and avoid hidden profiling.

## Public vs private methodology

This repository publishes public methodology, not the production engine.

Public:

- general structure;
- didactic pseudocode;
- fictional examples;
- public formulas;
- privacy principles;
- deontological safeguards.

Private:

- scoring weights;
- full questionnaires;
- complete production code;
- lead-routing logic;
- partner integrations;
- conversion UX;
- real datasets.
